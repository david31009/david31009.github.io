怎麼讓 google 搜尋到我的部落格 - 新增 sitemap & GA4 & adsense

發表於2024-07-17|更新於2024-08-07|Hexo 系列文章
|閱讀量:

前言

部落格架好之後，除了寫好文章，當然也要追蹤文章的流量、點擊率、觸及人數等。這篇文章主要是記錄如何產出 sitemap，並串接 GA4 以及 google adsense。

加入 google 搜尋引擎

所謂加入 google 搜尋引擎，就是要讓 google 能搜尋的到我的網站 ( 部落格 )，不是只靠貼網址才能找到。為了能讓 google 搜尋得到，就必須要有 sitemap 這個檔案，這是 google 爬蟲用來讀取網站的網站地圖，可以把它想成網站摘要的概念。

  1. 下載 Hexo 生成 sitemap 的工具，hexo 很方便，直接有生成 sitemap 的套件

    • npm install hexo-generator-sitemap --save
    • 於 _config.yaml 新增 sitemap 路徑

      • 網站部屬後會自動生成新的 sitemap，可輸入 網址/sitemap.xml 檢查，或是到 GitHub 上看有沒有 sitemap.xml 這個檔案

      • sitemap 的路徑可以自訂，一般會放在第一層 ( my-blog/sitemap )，可在 path 參數修改

      • tags 跟 categories 參數，決定是否要讓 sitemap 包含標籤頁跟分類頁。這邊建議不要，因為這些頁面都偏空洞，可能會讓 google 很難建立索引。

  2. 驗證網站擁有權及網域:

    • 到 Google Search Console: 把網頁加入 google 搜尋引擎的工具。

    • 這邊使用網址前置字元，驗證網域

    • 驗證網域前，需先驗證「網站所有權」，這邊使用 HTML 標記驗證，把 meta 資料塞入網站的 <head></head> 區塊中

    • 修改 _config.butterfly.yaml:

      • #Inject 部分可以新增 meta 資料到 <head></head> 中

      • 後來發現 #Verification 區塊可以直接加入 content id 更方便

    • 成功畫面

  3. 成功驗證網域後，可以進入網站資源，加入 sitemap

    • 到 Google Search Console 填入 sitemap.xml

    • 之後 sitemap 若有更新，google 爬蟲也會自動更新

加入 Google Analytics 4

  • 申請 Google Analytics 帳戶

    • 新增資料串流: 登入 GA，點選左下角 管理 > 資料收集與修改 > 資料串流 > 點選新增串流 > 網頁，接著把串流名稱 ( 可任意填 )、串流網址填好，點選新增

    • 獲取評估 ID: 新增完資料串流就會跳出評估 ID

  • 修改 _config.butterfly.yaml
    把評估 ID ( G 開頭的一串代碼 ) 填到 # Google Analytics 區塊中

  • 驗證

    • 檢查網頁原始碼

      • 打開 chrome dev tool ( 開發工具 )，切到 Element 頁籤，搜尋 gtag

    • 在 GA4 後台測試是否有埋 GA4 的 tracking id

      • 新增串流時可以順便測試 ( 這個頁面先不要關 )，正確安裝會亮綠勾勾

      • 如果不小心關閉上面的測試頁面，可以到資料串流 > 查看代碼操作說明 > 手動安裝

      • 點選測試: 出現綠勾勾代表成功埋入 ga4 的追蹤碼

加入 Google AdSense

加入 Google AdSense 的主要用意是： 可以出租網站的部分版位給 google 放廣告，然後 google 會給你一些費用，但是聽說很少。基於好奇心，我還是安裝看看，感覺不會太複雜。

  • 建立 Google AdSense 帳號

  • 完成三步驟 ( payments, ads, sites )

    • payment 資訊: 之後 google 會付錢給你的帳戶，需要驗證手機號碼

    • 設定 auto ads，自動生成廣告板位的預覽

    • 加入 Google AdSense 的 client id: 也是一串 ID 要埋設在網站中

      • 修改 _config.butterfly.yaml

        • enable 設定成 true

        • 新增 client id: ca-pub 開頭的代碼

      • 驗證

        • 查看網站原始碼: chrome dev tool 搜尋 adsbygoogle

        • 用 google ad sense 後台驗證

新增 ads.txt

  1. 用意: 列出哪些廣告商 ( 如 Google Adsense ) 有被我的網站授權，也就是說，有被我列出的廣告商，才可以用使我網站的版位投放 ( 賣 ) 廣告。
  2. 目的: 增強廣告市場的透明度和減少欺詐行為，也能夠賺取更多廣告收入。

  • 以下是來自 Google 官方說明 的文字:

    授權數位賣方 ( ads.txt ) 是一項 IAB Tech Lab 計劃，旨在協助您僅透過認定的授權賣方（如 AdSense）銷售您的數位廣告資源。

    建立自己的 ads.txt 檔案後，您可以更好地掌控哪些賣方有權銷售您網站上的廣告空間，並可防止向廣告客戶展示仿冒廣告資源。

    使用 ads.txt 並非強制性要求，但我們強烈建議您使用。它可以幫助買家識別仿冒廣告資源，並可以幫助您獲得更多廣告客戶支出，而這些支出原本可能會流向仿冒廣告資源

  • 到 Google Adsense 複製 ads.txt 內容。
    登入 Google Adsense > Site > 點選網站網址 > 複製內容

  • 在 source/ 下新增 ads.txt 檔

  • 部屬後，可以在 網域/ads.txt 檢查看看是否有新增成功
